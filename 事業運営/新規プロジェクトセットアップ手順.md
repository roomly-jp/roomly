# 新規プロジェクト セットアップ手順

SaaS プロジェクトを新規作成するときの標準手順。

---

## 1. ドメイン取得（お名前.com）

- お名前.comで希望ドメインを取得
- 管理画面 → DNS設定で以下を追加：
  - `example.jp` → Aレコード: `76.76.21.21`（Vercel）
  - `www.example.jp` → CNAME: `cname.vercel-dns.com`

## 2. メール設定（Google Workspace）

- https://workspace.google.com から Business Starter（年額プラン）を契約
- お名前.com DNS設定で Google の MX レコードを追加（Workspace の案内に従う）
- 以下のメールアドレスを作成：

| アドレス | 用途 | 使用箇所 |
|---------|------|---------|
| `admin@` | 管理者用（サービス登録・DNS確認等） | 各種外部サービスの管理者メール |
| `contact@` | お問い合わせ窓口 | HP問い合わせセクション・料金問い合わせ |
| `support@` | ユーザーサポート | アプリ内のヘルプ・サポートリンク |
| `info@` | 一般的な情報提供 | 名刺・公式案内等 |
| `billing@` | 請求・決済関連 | 請求書・決済通知の送信元 |
| `noreply@` | システム自動送信 | パスワードリセット・通知メール（Supabase Auth等） |

## 3. GitHub Organization 作成

- https://github.com/organizations/new でOrg作成
- リポジトリを作成（例: roomly, hp, lp）
- 各プロジェクトを init → push：
  ```bash
  cd <project-dir>
  git init && git add -A && git commit -m "初期コミット"
  git branch -M main
  git remote add origin https://github.com/<org>/<repo>.git
  git push -u origin main
  ```
- gh CLI を使う場合は `gh auth login --web` → `gh auth setup-git` で認証設定

## 4. Supabase プロジェクト作成

- https://supabase.com でプロジェクト作成
- SQL Editor でマイグレーションを実行（`supabase/migrations/` のSQLファイル）
- 以下の値をメモ（後でVercelに設定）：
  - Settings → API → **Project URL**
  - Settings → API → **anon public キー**

## 5. Vercel デプロイ

- https://vercel.com で GitHub アカウントでサインアップ
- GitHub Org を連携（Add GitHub Org or Account）
- メインアプリのリポジトリをImport → Deploy
- 環境変数を設定（Settings → Environment Variables）：
  | 変数名 | 値 |
  |--------|-----|
  | `NEXT_PUBLIC_SUPABASE_URL` | SupabaseのProject URL |
  | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabaseのanon publicキー |
- Settings → Domains でカスタムドメインを追加（DNS設定済みならSSL自動発行）

## 6. サブドメイン設計（例）

| ドメイン | 用途 | プロジェクト |
|---------|------|------------|
| `example.jp` | メインアプリ | roomly/ |
| `www.example.jp` | HP or リダイレクト | hp/ |
| `lp.example.jp` | ランディングページ | lp/ |

## 7. ローカルからSupabase接続

- プロジェクトに `@supabase/supabase-js` と `@supabase/ssr` をインストール
- `.env.local` に接続情報を設定（`.gitignore` で除外されていること確認）：
  ```
  NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
  NEXT_PUBLIC_SUPABASE_ANON_KEY=sb_publishable_xxxxx
  ```
- `src/lib/supabase.ts` にクライアント作成関数を用意
- 接続テスト：Node.jsスクリプトでテーブルにクエリを実行して確認

## 8. メール送信設定（Resend）

### アカウント作成 & ドメイン認証

1. https://resend.com でサインアップ
2. Domains → Add Domain → `example.jp` を追加
3. 表示されるDNSレコード（SPF: TXT、DKIM: CNAME × 2〜3）をお名前.comに追加
4. Resend で Verify をクリック → 認証完了
5. API Keys → Create API Key → キーを取得

### 環境変数

- `.env.local` に `RESEND_API_KEY=re_xxxxx` を追加
- Vercel にも同じ環境変数を設定（Settings → Environment Variables）

### コード構成

- `src/lib/email.ts` — メール送信ユーティリティ（`sendEmail()` 関数）
- `src/app/api/email/send/route.ts` — メール送信 API エンドポイント
- `src/app/api/email/test/route.ts` — テスト送信（開発環境のみ）

### テスト方法

```bash
# ローカルで dev サーバー起動後
curl "http://localhost:3000/api/email/test?to=your@email.com"
```

### 無料枠

- 月3,000通まで無料
- 1日100通の送信制限

## 9. SEO・マーケティング設定

### Google Analytics (GA4)

1. https://analytics.google.com でプロパティ作成
2. 測定ID（`G-XXXXXXXXXX`）を取得
3. Next.jsの場合 — `layout.tsx` に `next/script` で設定：
   ```tsx
   import Script from "next/script";
   // <head>内に配置
   <Script src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX" strategy="afterInteractive" />
   <Script id="gtag-init" strategy="afterInteractive">
     {`window.dataLayer = window.dataLayer || [];
       function gtag(){dataLayer.push(arguments);}
       gtag('js', new Date());
       gtag('config', 'G-XXXXXXXXXX');`}
   </Script>
   ```
4. Vite/静的サイトの場合 — `index.html` の `<head>` に直接 `<script>` タグを記述

### Google Search Console

1. https://search.google.com/search-console でプロパティ追加
2. 所有権確認方法を選択：
   - **DNS TXTレコード**: お名前.comでTXTレコードを追加（推奨、全サブドメインまとめて確認可能）
   - **HTMLタグ**: `<meta name="google-site-verification" content="xxxxx" />` をheadに追加
3. サイトマップ送信: `https://example.jp/sitemap.xml` を登録

### sitemap.xml

- **Next.js**: `src/app/sitemap.ts` で動的生成（記事追加時に自動反映）
  ```ts
  import type { MetadataRoute } from "next";
  export default function sitemap(): MetadataRoute.Sitemap {
    return [{ url: "https://example.jp", lastModified: new Date(), priority: 1.0 }];
  }
  ```
- **Vite/静的サイト**: `public/sitemap.xml` に静的ファイルとして配置

### robots.txt

- **Next.js**: `src/app/robots.ts` で生成
- **Vite/静的サイト**: `public/robots.txt` に配置
- 内容: `User-agent: *`, `Allow: /`, `Sitemap: https://example.jp/sitemap.xml`

### コラム（SEO記事）

- Next.jsプロジェクト（HP）に `/column` ページを作成
- 記事データは `src/lib/blog.ts` にTS配列として管理
- 各記事に OGP メタデータ、パンくず、関連記事リンク、CTA を設置
- `sitemap.ts` で記事URLも自動で含める

### ファビコン

- SVGファビコン推奨（軽量・スケーラブル）
- Next.js: `src/app/icon.svg`（favicon.icoは削除）
- Vite: `public/favicon.svg`（`index.html` で `<link rel="icon" type="image/svg+xml" href="/favicon.svg" />`）

## 10. 確認チェックリスト

- [ ] ドメインでアプリにアクセスできる
- [ ] SSL（https）が有効
- [ ] Supabase接続が動作する
- [ ] メール送受信できる（Gmail）
- [ ] Resendドメイン認証済み
- [ ] Resendテスト送信が届く
- [ ] GA4でアクセスが計測されている
- [ ] Search Consoleで所有権が確認済み
- [ ] sitemap.xmlがSearch Consoleに送信済み
- [ ] robots.txtが正しく配信されている
- [ ] ファビコンが表示されている
- [ ] OGP（Open Graph）が設定されている（SNSシェア時のプレビュー）
