# Roomly PRD — Ralph Loop用タスク一覧

## 概要
賃貸管理会社向け物件・入居者・契約・家賃管理SaaS。
現状: UI実装済み（9画面）、DB未デプロイ、認証モック状態。
このPRDはRalph Loopで自律実行するためのタスク一覧。

- 横断要件（セキュリティ・設計原則・パフォーマンス・DB効率）を全タスクで遵守する

## 横断要件（全フェーズ共通）

### セキュリティ（OWASP Top 10準拠）
- SQLインジェクション防止: Supabase SDKパラメータバインディング必須（生SQL禁止）
- XSS防止: Reactデフォルト挙動（`dangerouslySetInnerHTML`禁止）
- 認証・認可: Supabase Auth + RLS（company_id）
- CSRF対策: SameSite Cookie + Supabase Auth
- 機密情報: `.env` / APIキーのコミット禁止
- 個人情報保護: 入居者データはRLSでテナント分離

### 設計原則
- DDD/クリーンアーキテクチャ: UI → Application → Domain → Infrastructure
- ビジネスロジック（家賃計算・送金計算・滞納判定等）はドメイン層に集約

### パフォーマンス
- Web: Core Web Vitals（LCP < 2.5s, CLS < 0.1, INP < 200ms）
- API: Supabaseクエリ200ms以内
- ダッシュボード: 初期表示2秒以内

### DB効率
- インデックス: `company_id`・`created_at`・外部キー・検索カラム
- N+1回避: JOINまたは一括取得（ループ内クエリ禁止）
- SELECT指定: 必要カラムのみ（`select(*)` 禁止）
- ページネーション: 20件/ページ
- 集計: DBレベル（フロントで全件計算しない）
